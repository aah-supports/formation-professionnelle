Python Niveau II — Plan de cours détaillé

Durée : 2 jours (14h) — progression continue, exercices à difficulté croissante.

Jour 1 — Objet et décorateurs

CréneauProgressionContenuExercice guidéCritère de réussite
09:00–10:30 Base Refonte d’un script en objets : responsabilités, attributs, méthodes, dataclasses. Modéliser une entité métier (Commande, Client, Produit). Chaque classe a une responsabilité claire.
10:45–12:30 Intermédiaire Composition vs héritage, interfaces implicites, contrats de méthodes, introduction au data model Python. Refactor d’un héritage excessif en composition. Couplage réduit, meilleure lisibilité.
13:30–15:00 Intermédiaire+ Décorateurs : wraps, paramètres, logging, validation. Créer @validate_input et @log_duration. Décorateurs réutilisables et testables.
15:15–17:00 Consolidation Atelier de synthèse J1 : architecture orientée objet + décorateurs. Mini-moteur de traitement (entrée/sortie + règles métier). Code cohérent, sans duplication majeure.

Jour 2 — Coroutines et async/await

CréneauProgressionContenuExercice guidéCritère de réussite
09:00–10:30 Base Event loop, coroutine, await, create_task, gather. Paralléliser 20 appels I/O simulés. Temps d’exécution significativement réduit.
10:45–12:30 Intermédiaire Gestion des erreurs async : timeout, annulation, retry, backoff. Rendre tolérant un lot d’appels API instables. Aucun crash global, erreurs tracées proprement.
13:30–15:00 Intermédiaire+ Limiter la concurrence (Semaphore), éviter surcharge et throttling. Respecter une limite de 5 appels concurrents. Débit stable, pas d’explosion d’erreurs.
15:15–17:00 Avancé Projet final : service asynchrone complet orienté objet. Assembler modèle, décorateurs, orchestration async. Livrable exécutable + quiz + recommandations.

Bloc théorique transverse (niveau II)

  • Méthodes magiques : implémenter correctement __repr__, __len__, __iter__, __eq__ selon les besoins métier.
  • Cycle de vie d'objet : rôle de __init__, __new__, immutabilité et invariants.
  • MRO et héritage multiple : lecture et impact sur la maintenabilité.
  • Conception API Pythonique : exceptions explicites, méthodes courtes, noms cohérents, docstrings ciblées.
  • Async en production : backpressure, cancellation, retry idempotent, observabilité minimale.

import functools

def positive_amount(fn):
    @functools.wraps(fn)
    def wrapper(amount, *args, **kwargs):
        if amount <= 0:
            raise ValueError("amount doit être positif")
        return fn(amount, *args, **kwargs)
    return wrapper
import asyncio

sem = asyncio.Semaphore(5)

async def safe_call(call):
    async with sem:
        return await asyncio.wait_for(call(), timeout=2)
class Money:
    def __init__(self, amount: float, currency: str = "EUR") -> None:
        self.amount = amount
        self.currency = currency

    def __repr__(self) -> str:
        return f"Money(amount={self.amount}, currency='{self.currency}')"

    def __add__(self, other: "Money") -> "Money":
        if self.currency != other.currency:
            raise ValueError("devise incompatible")
        return Money(self.amount + other.amount, self.currency)
