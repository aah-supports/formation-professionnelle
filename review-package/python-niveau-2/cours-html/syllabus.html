Python Niveau II — Syllabus

POO avancée • Décorateurs • Coroutines • async/await

Durée recommandée : 2 jours (14h) — niveau intermédiaire.

Objectif général

Transformer une pratique Python « script » en une pratique « application » : code structuré, réutilisable, testable et capable de gérer des I/O concurrentes proprement.

Progression pédagogique (du simple au complexe)

Étape 1 — Structurer le codePasser d'un script procédural à un modèle orienté objet clair.
Étape 2 — Factoriser les comportements transversesAppliquer des décorateurs pour logs, validation, retry.
Étape 3 — Concurrence asynchroneComprendre l'event loop et piloter des coroutines.
Étape 4 — Fiabiliser en conditions réellesGérer timeouts, annulations, limites de concurrence et erreurs.
Étape 5 — IntégrerAssembler objet + décorateurs + async dans un mini-service cohérent.

Définitions essentielles

  • Classe : plan de construction d'objets portant état et comportements.
  • Invariant : règle qui doit rester vraie pendant la vie d'un objet.
  • Décorateur : fonction qui enrichit une autre fonction sans modifier son code interne.
  • Coroutine : fonction asynchrone suspendable/reprenable avec await.
  • Event loop : ordonnanceur des tâches asynchrones.

Notions théoriques supplémentaires

  • Data model Python : protocole d'objets piloté par des méthodes spéciales (dunder).
  • Méthodes magiques : __repr__, __str__, __len__, __iter__, __eq__, __lt__, __hash__, __call__.
  • Protocole de contexte : __enter__ / __exit__ pour gérer proprement les ressources.
  • Descriptors et properties : encapsulation fine via @property et validation d'accès.
  • MRO : ordre de résolution des méthodes en héritage multiple.
  • Protocoles de typage : interfaces structurelles avec typing.Protocol.

Extraits de code de référence

POO + invariants

from dataclasses import dataclass

@dataclass
class Compte:
    id: str
    solde: float

    def __post_init__(self) -> None:
        if self.solde < 0:
            raise ValueError("solde invalide")

Méthodes magiques (dunder)

class Panier:
    def __init__(self, items: list[str]) -> None:
        self.items = items

    def __len__(self) -> int:
        return len(self.items)

    def __iter__(self):
        return iter(self.items)

    def __repr__(self) -> str:
        return f"Panier(items={self.items!r})"

Décorateur

import functools

def log_call(fn):
    @functools.wraps(fn)
    def wrapper(*args, **kwargs):
        print(f"call {fn.__name__}")
        return fn(*args, **kwargs)
    return wrapper

Async/await

import asyncio

async def fetch_one(i: int) -> str:
    await asyncio.sleep(0.2)
    return f"ok-{i}"

async def main() -> None:
    results = await asyncio.gather(*(fetch_one(i) for i in range(5)))
    print(results)

Compétences visées

  • Concevoir des classes métier robustes (invariants, encapsulation, composition).
  • Écrire des décorateurs lisibles et sûrs sans effets de bord.
  • Orchestrer des appels I/O concurrents avec asyncio.
  • Diagnostiquer les anti-patterns (blocage event loop, couplage fort, exceptions silencieuses).
  • Produire un code maintenable avec tests ciblés.

Public cible et prérequis

  • Public : développeurs Python juniors/intermédiaires, profils data techniques.
  • Prérequis : fonctions, modules, exceptions, manipulations de fichiers et notions de tests.
