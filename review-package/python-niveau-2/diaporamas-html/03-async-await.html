Chapitre 3 — Coroutines et async/await

Concevoir des flux I/O concurrents robustes et observables.

Notions fondamentales

  • Coroutine : fonction suspendable avec await.
  • Task : coroutine planifiée dans l'event loop.
  • Event loop : ordonnanceur coopératif.
  • Concurrence : chevauchement d'I/O, pas parallélisme CPU.

Exécution de base

import asyncio

async def fetch_one(i: int) -> str:
    await asyncio.sleep(0.2)
    return f"ok-{i}"

async def main() -> None:
    results = await asyncio.gather(*(fetch_one(i) for i in range(5)))
    print(results)

asyncio.run(main())

Structured concurrency avec TaskGroup

import asyncio

async def worker(name: str):
    await asyncio.sleep(0.1)
    return name

async def main():
    async with asyncio.TaskGroup() as tg:
        tg.create_task(worker("A"))
        tg.create_task(worker("B"))

Si une tâche échoue, le groupe gère les annulations des autres tâches.

Timeout et annulation

import asyncio

async def call_remote():
    await asyncio.sleep(2)
    return "done"

async def guarded_call():
    try:
        return await asyncio.wait_for(call_remote(), timeout=0.5)
    except asyncio.TimeoutError:
        return "timeout"

Backpressure avec Semaphore

import asyncio

sem = asyncio.Semaphore(10)

async def fetch(url: str):
    async with sem:
        await asyncio.sleep(0.05)
        return {"url": url, "status": 200}

Pipeline producteur/consommateur

import asyncio

async def producer(q: asyncio.Queue):
    for i in range(50):
        await q.put(i)
    await q.put(None)

async def consumer(q: asyncio.Queue):
    while True:
        item = await q.get()
        if item is None:
            break

Gestion des erreurs multi-tâches

  • gather() propage la première exception par défaut.
  • return_exceptions=True permet de collecter les échecs.
  • Avec TaskGroup, raisonner en « unité de travail ».
  • Toujours tracer contexte, tentative et durée.

Async generator

import asyncio

async def stream_rows():
    for i in range(3):
        await asyncio.sleep(0.1)
        yield {"row": i}

async def main():
    async for row in stream_rows():
        print(row)

Interop avec code bloquant

import asyncio
import time

def blocking_call(x: int) -> int:
    time.sleep(0.2)
    return x * 2

async def run_blocking(x: int) -> int:
    return await asyncio.to_thread(blocking_call, x)

Quand ne pas utiliser async

  • Tâches CPU lourdes sans I/O dominant.
  • Scripts simples séquentiels avec faible latence.
  • Équipe non outillée pour debug async en production.
  • Pipeline où le goulot est essentiellement SQL ou réseau externe saturé.

Patterns de fiabilité

  • Retry idempotent avec backoff exponentiel.
  • Bulkhead: limiter la concurrence par type de ressource.
  • Timeout explicite sur chaque dépendance externe.
  • Cancellation propre pour éviter tâches zombies.

Observabilité minimale async

import logging
logger = logging.getLogger(__name__)

logger.info(
    "api_call",
    extra={"endpoint": "/orders", "attempt": 2, "latency_ms": 183}
)

Checklist de revue async

  • Un timeout est-il défini pour chaque I/O externe ?
  • Le niveau de concurrence est-il borné ?
  • Les erreurs partielles sont-elles gérées sans crash global ?
  • Les logs permettent-ils de diagnostiquer une saturation ?

Récapitulatif

  • Async améliore le débit I/O si la conception reste disciplinée.
  • TaskGroup, timeouts et backpressure sont les briques clés.
  • La robustesse dépend autant de l'observabilité que du code.
