Bloc théorique transverse (niveau III)

  • Design de pipeline data : ingestion, standardisation, enrichissement, publication.
  • Qualité statistique : outliers, biais de sélection, stabilité des agrégats.
  • Fiabilité opérationnelle : idempotence, reprise sur incident, journalisation des étapes.
  • Performance data : vectorisation, limitation des copies, chunking sur gros volumes.
  • Traçabilité métier : hypothèses explicites et définition versionnée des KPI.

def quality_report(df):
    return {
        "rows": len(df),
        "na_rate": (df.isna().sum() / len(df)).to_dict(),
        "duplicates": int(df.duplicated().sum())
    }
