Chapitre 1 — Fondations NumPy

Construire des traitements numériques fiables et performants.

Objectifs d'apprentissage

  • Comprendre mémoire, shape, dtype et axes.
  • Vectoriser sans perdre la lisibilité métier.
  • Éviter les copies cachées et les erreurs de broadcasting.
  • Mesurer les gains avec des benchmarks simples.

Structure interne d'un ndarray

import numpy as np

arr = np.array([[1, 2, 3], [4, 5, 6]], dtype=np.float64)
print(arr.shape)    # (2, 3)
print(arr.ndim)     # 2
print(arr.dtype)    # float64
print(arr.strides)  # déplacement mémoire par axe

Pourquoi le dtype compte

  • float64 : précision, coût mémoire plus élevé.
  • float32 : plus compact, attention aux erreurs d'arrondi.
  • int64 : identifiants et compteurs robustes.
  • object : éviter pour les calculs vectorisés.

Vectorisation vs boucle

import numpy as np

x = np.random.rand(2_000_000)
# boucle Python lente
# for i in range(len(x)): ...

# vectorisé
z = (x - x.mean()) / x.std()

Broadcasting: règles minimales

  • Comparer dimensions de droite à gauche.
  • Deux dimensions sont compatibles si égales ou si l'une vaut 1.
  • En cas d'incompatibilité: erreur explicite de shape.
sales = np.array([[100, 120, 140], [80, 95, 110]])
coef = np.array([1.0, 0.9, 1.1])
adjusted = sales * coef

Indexation avancée

m = np.arange(12).reshape(3, 4)

sub = m[:, [0, 2]]      # fancy indexing
mask = m % 2 == 0
pairs = m[mask]         # masque booléen

print(sub)
print(pairs[:4])

Copy vs view

x = np.array([10, 20, 30, 40])
view = x[1:3]          # vue
clone = x[1:3].copy()  # copie réelle

view[0] = 999
print(x)      # modifié
print(clone)  # indépendant

Reshape, transpose, flatten

x = np.arange(12)
m = x.reshape(3, 4)
mt = m.T
flat = m.ravel()  # vue si possible

print(m.shape, mt.shape, flat.shape)

Agrégations robustes

m = np.array([[2, 4, 8], [1, 3, 9]])
row_sum = m.sum(axis=1)
col_mean = m.mean(axis=0)
col_std = m.std(axis=0, ddof=1)

print(row_sum, col_mean, col_std)

NumPy random et reproductibilité

rng = np.random.default_rng(seed=42)
sample = rng.normal(loc=100, scale=15, size=5)
print(sample)
  • Utiliser un seed permet des tests et comparaisons stables.

Algèbre linéaire de base

A = np.array([[1., 2.], [3., 4.]])
b = np.array([5., 6.])
x = np.linalg.solve(A, b)
print(x)

Benchmark minimal

from time import perf_counter

x = np.random.rand(500_000)
t0 = perf_counter()
_ = np.sqrt(x) + np.log1p(x)
print(f"{(perf_counter()-t0)*1000:.1f} ms")

Pièges courants en production

  • Confondre axes 0/1 et publier des agrégats faux.
  • Forcer des conversions répétées de type en pipeline.
  • Créer des copies massives involontaires.
  • Ignorer les NaN avant des calculs statistiques.

Récapitulatif

  • NumPy est la base de la performance data en Python.
  • dtype, axes et mémoire sont des notions non négociables.
  • Vectoriser oui, mais toujours avec contrôles de qualité.
