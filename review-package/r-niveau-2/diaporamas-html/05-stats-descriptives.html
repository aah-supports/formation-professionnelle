Module 5 — Qualité des données

Contrôler avant de calculer

1 / 10

Pourquoi ce module ?

  • Un indicateur faux peut conduire à une mauvaise décision
  • Les erreurs de clé sont souvent invisibles
  • La qualité doit être mesurée, pas supposée
  • Chaque extraction doit embarquer des contrôles
2 / 10

Profil qualité rapide

profil_qc <- df |>
  summarise(across(
    everything(),
    list(
      na_n = ~ sum(is.na(.x)),
      na_pct = ~ round(mean(is.na(.x)) * 100, 2),
      n_distinct = ~ n_distinct(.x)
    )
  ))

Contrôles métier

  • Montants négatifs interdits (hors avoirs)
  • Dates futures non autorisées
  • Codes segment hors référentiel
  • Volumes anormaux par région
5 / 10

Réconciliation base vs Excel

reco <- ca_base |>
  full_join(ca_excel, by = c("mois", "region")) |>
  mutate(
    ca_base = coalesce(ca_base, 0),
    ca_excel = coalesce(ca_excel, 0),
    ecart = ca_base - ca_excel,
    ecart_pct = if_else(ca_excel == 0, NA_real_, ecart / ca_excel)
  )
