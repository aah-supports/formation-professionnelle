Exercice 1.2 : Première requête dbplyr [Difficulté: Intermédiaire]

Énoncé : À partir de la table ventes :

  1. Créer un objet distant tbl(con, "ventes")
  2. Filtrer les lignes avec date >= "2025-01-01"
  3. Garder uniquement date, region, prix_net
  4. Calculer le CA total par région
  5. Afficher le SQL généré et collecter le résultat final
Solution 
library(dplyr)
library(dbplyr)

ventes_tbl <- tbl(con, "ventes")

ca_region <- ventes_tbl |>
  filter(date >= "2025-01-01") |>
  select(date, region, prix_net) |>
  group_by(region) |>
  summarise(ca_total = sum(prix_net, na.rm = TRUE), .groups = "drop")

show_query(ca_region)
ca_region |> collect()

Exercice 1.3 : Limiter les volumes [Difficulté: Intermédiaire]

Énoncé : Pourquoi faut-il éviter collect() trop tôt ? Vérifiez sur cet exemple :

  1. Version A : collect() puis filter()
  2. Version B : filter() côté base puis collect()
  3. Comparer le nombre de lignes ramenées
Solution 
# Version A (moins efficace)
a <- tbl(con, "ventes") |>
  collect() |>
  filter(date >= as.Date("2025-01-01"))

# Version B (recommandée)
b <- tbl(con, "ventes") |>
  filter(date >= "2025-01-01") |>
  collect()

nrow(a)
nrow(b)

Exercice 2.2 : Nettoyer un export messy [Difficulté: Intermédiaire]

Énoncé : Le fichier objectifs.xlsx contient 2 lignes d'entête et des montants au format texte.

  1. Lire la feuille Objectifs en sautant 2 lignes
  2. Convertir objectif_ca en numérique
  3. Supprimer les lignes où region est vide
Solution 
library(stringr)

objectifs <- read_excel("objectifs.xlsx", sheet = "Objectifs", skip = 2) |>
  clean_names() |>
  mutate(
    objectif_ca = str_replace_all(objectif_ca, "[ €]", "") |>
      str_replace(",", ".") |>
      as.numeric()
  ) |>
  filter(!is.na(region), region != "")

Exercice 2.3 : Passer d'un format large à tidy [Difficulté: Intermédiaire]

Énoncé : Vous avez des colonnes jan_2025, fev_2025, mar_2025 dans objectifs_wide.

  1. Transformer en format long avec pivot_longer()
  2. Séparer la colonne mois/année
  3. Créer une date du premier jour du mois
Solution 
library(tidyr)

objectifs_tidy <- objectifs_wide |>
  pivot_longer(
    cols = jan_2025:mar_2025,
    names_to = "periode",
    values_to = "objectif_ca"
  ) |>
  separate(periode, into = c("mois", "annee"), sep = "_") |>
  mutate(date_mois = as.Date(paste0("01-", mois, "-", annee), format = "%d-%b-%Y"))

Module 3 : Requêtes avancées I

Exercice 3.1 : Jointures multi-tables [Difficulté: Intermédiaire]

Énoncé : Construire une table analytique en joignant :

  • ventes avec clients via client_id
  • puis avec produits via produit_id

Ne conserver que les colonnes utiles : date, region, segment, famille, quantite, prix_net.

Solution 
ventes_tbl <- tbl(con, "ventes")
clients_tbl <- tbl(con, "clients")

base_analyse <- ventes_tbl |>
  left_join(clients_tbl, by = "client_id") |>
  left_join(produits, by = "produit_id") |>
  select(date, region, segment, famille, quantite, prix_net)

base_analyse |> collect() |> glimpse()

Exercice 3.2 : Détecter les références absentes [Difficulté: Avancé]

Énoncé : Trouvez les produit_id présents dans les ventes mais absents du catalogue Excel.

Indice : Utilisez anti_join().

produits_absents <- ventes_tbl |>

Exercice 3.3 : Top 5 clients par région [Difficulté: Avancé]

Énoncé : Calculez le chiffre d'affaires par client et région, puis gardez les 5 meilleurs clients de chaque région.

Solution 
top_clients <- ventes_tbl |>
  group_by(region, client_id) |>
  summarise(ca = sum(prix_net, na.rm = TRUE), .groups = "drop") |>
  group_by(region) |>
  mutate(rang = min_rank(desc(ca))) |>
  filter(rang <= 5) |>
  arrange(region, rang) |>
  collect()

top_clients

Module 4 : Requêtes avancées II et automatisation

Exercice 4.1 : Calcul d'évolution mensuelle [Difficulté: Avancé]

Énoncé : Créez un indicateur d'évolution mensuelle du CA par région avec lag().

  1. Agréger le CA mensuel
  2. Créer ca_mois_prec et variation_pct
  3. Trier par région puis mois
Solution 
evo_mensuelle <- ventes_tbl |>
  mutate(mois = substr(date, 1, 7)) |>
  group_by(region, mois) |>
  summarise(ca = sum(prix_net, na.rm = TRUE), .groups = "drop") |>
  group_by(region) |>
  arrange(mois, .by_group = TRUE) |>
  mutate(
    ca_mois_prec = lag(ca),
    variation_pct = (ca - ca_mois_prec) / ca_mois_prec
  ) |>
  ungroup() |>
  collect()

Exercice 4.2 : Fonction d'extraction paramétrée [Difficulté: Avancé]

Énoncé : Écrire une fonction extraire_kpi(con, date_debut, date_fin, region = NULL) qui :

  • Filtre les ventes sur la période
  • Filtre la région si fournie
  • Retourne CA, nombre de commandes, panier moyen
Solution 
extraire_kpi <- function(con, date_debut, date_fin, region = NULL) {
  req <- tbl(con, "ventes") |>
    filter(date >= date_debut, date <= date_fin)

  if (!is.null(region)) {
    req <- req |> filter(region == !!region)
  }

  req |>
    summarise(
      ca_total = sum(prix_net, na.rm = TRUE),
      nb_commandes = n_distinct(id_commande),
      panier_moyen = ca_total / nb_commandes
    ) |>
    collect()
}

extraire_kpi(con, "2025-01-01", "2025-03-31", "Nord")

Exercice 4.3 : Import robuste multi-fichiers [Difficulté: Avancé]

Énoncé : Importez tous les fichiers .xlsx du dossier exports/ et capturez les erreurs sans interrompre le traitement.

Solution 
library(purrr)

safe_read_excel <- possibly(
  ~ read_excel(.x, sheet = "Data") |> clean_names(),
  otherwise = NULL
)

imports <- list.files("exports", pattern = "\\.xlsx$", full.names = TRUE) |>
  set_names() |>
  map(safe_read_excel)

resultat <- imports |>
  compact() |>
  list_rbind(names_to = "source_fichier")

resultat
