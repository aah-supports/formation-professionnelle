Module 6 — Structuration du code et validation

Passer d'un script unique à un mini-projet Python propre.

Objectifs

  • Découper un script en fonctions simples.
  • Organiser un projet avec des modules lisibles.
  • Valider les entrées et gérer les erreurs.
  • Préparer une exécution reproductible.

Découpage fonctionnel

def load_rows(path: str):
    ...


def clean_rows(rows):
    ...


def build_report(rows):
    ...


def main():
    rows = load_rows("data.csv")
    rows = clean_rows(rows)
    print(build_report(rows))

Convention de nommage

  • Fonctions en snake_case.
  • Variables métier explicites.
  • Constantes en MAJUSCULES.
  • Éviter les abréviations ambiguës.

Validation des entrées

def parse_amount(value: str) -> float:
    amount = float(value)
    if amount < 0:
        raise ValueError("montant négatif")
    return amount


def parse_date(value: str) -> str:
    if len(value) != 10:
        raise ValueError("date invalide")
    return value

Gestion d'erreurs propre

def run(path: str) -> None:
    try:
        rows = load_rows(path)
        report = build_report(rows)
        print(report)
    except FileNotFoundError:
        print("fichier introuvable")
    except ValueError as err:
        print(f"donnée invalide: {err}")

Séparer configuration et logique

INPUT_PATH = "data/ventes.csv"
OUTPUT_PATH = "out/report.json"


def main() -> None:
    rows = load_rows(INPUT_PATH)
    result = build_report(rows)
    save_json(OUTPUT_PATH, result)

Exemple de module utilitaire

# utils_dates.py

def to_month_key(date_iso: str) -> str:
    # "2026-04-21" -> "2026-04"
    return date_iso[:7]

Un module = un fichier avec une responsabilité claire.

Tests manuels systématiques

  • Cas nominal (données valides).
  • Cas limite (liste vide, zéro, texte vide).
  • Cas erreur (fichier absent, format invalide).
  • Comparer sortie obtenue vs sortie attendue.

Checklist qualité

  • Fonctions courtes et intention explicite.
  • Pas de duplication de logique.
  • Messages d'erreur compréhensibles.
  • Entrées/sorties documentées.

Récapitulatif

  • La structuration évite le script monolithique fragile.
  • Validation + erreurs explicites sécurisent l'exécution.
  • Un code clair est plus rapide à maintenir.
