Python Niveau IV — Plan de cours détaillé

Durée : 3 jours (21h) — parcours avancé architecture, qualité, performance et delivery.

Jour 1 — Architecture et design orienté production

CréneauProgressionContenuCode / livrableCritère de réussite
09:00–10:30 Base avancée Architecture modulaire : couches, frontières, dépendances. Cartographie d'une base existante. Découpage clair et justifié.
10:45–12:30 Intermédiaire Ports/adaptateurs, inversion de dépendances, composition root. Interfaces Protocol pour domaine. Domaine testable sans infrastructure réelle.
13:30–15:00 Intermédiaire+ Design des cas d'usage et gestion des erreurs métier. Service applicatif refactoré. Flux métier lisible, erreurs explicites.
15:15–17:00 Consolidation Atelier de refonte architecture sur module critique. Blueprint « avant / après ». Architecture revue et validée collectivement.

Jour 2 — Qualité, typing, tests et observabilité

CréneauProgressionContenuCode / livrableCritère de réussite
09:00–10:30 Intermédiaire Typing strict, contrats d'API interne, cohérence des données. Module typé de bout en bout. Mypy vert, signatures cohérentes.
10:45–12:30 Intermédiaire+ Tests unitaires et d'intégration, cas limites et non-régression. Suite de tests ciblée. Comportements critiques sécurisés.
13:30–15:00 Intermédiaire+ Observabilité : logs structurés, métriques, traces de diagnostic. Instrumentation d'un service. Incident reproductible et analysable rapidement.
15:15–17:00 Consolidation Chaîne qualité complète locale et CI. Pipeline lint + typing + tests. Merge bloqué en cas de régression.

Jour 3 — Performance et industrialisation

CréneauProgressionContenuCode / livrableCritère de réussite

13:30–15:00 Projet Delivery : versioning, release progressive, rollback, runbook. Pipeline CI/CD prêt production. Scénario de déploiement sécurisé validé.
15:15–17:00 Clôture Revue finale, quiz, feuille de route technique. Plan d'amélioration équipe. Capacité à piloter un projet Python production.

Bloc théorique transverse (niveau IV)

  • Architecture pilotée domaine : entités, services de domaine, invariants.
  • Contraintes runtime Python : GIL, coût des abstractions, arbitrage sync/async/process.
  • Qualité au niveau système : critères d'acceptation non fonctionnels (latence, robustesse, sécurité).
  • Observabilité avancée : corrélation logs/traces, budget d'erreurs, alertes actionnables.
  • Delivery engineering : fréquence de déploiement, lead time, taux d'échec, MTTR.

Repères de code du niveau IV

from typing import Protocol

class Repo(Protocol):
    def get(self, key: str): ...

def process(repo: Repo, key: str):
    item = repo.get(key)
    if item is None:
        raise ValueError("item introuvable")
    return item
class CircuitBreaker:
    def __init__(self, threshold: int = 3):
        self.threshold = threshold
        self.failures = 0

    def record(self, ok: bool) -> None:
        self.failures = 0 if ok else self.failures + 1

    @property
    def open(self) -> bool:
        return self.failures >= self.threshold
