  2. Installation de Python (15 min)

    • Téléchargement depuis python.org
    • Vérification de l'installation : python --version
    • Le gestionnaire de paquets pip

  3. Les environnements de développement (20 min)

    • VS Code : installation, extensions Python, terminal intégré
    • Jupyter Notebook : installation via pip, cellules de code et de texte
    • Autres options : PyCharm, IDLE (mention rapide)

  4. L'interpréteur interactif (20 min)

    • Python comme calculatrice : opérations arithmétiques
    • Affectation avec =
    • Afficher avec print() et f-strings
    • L'historique des commandes

  5. Les scripts Python (10 min)

    • Créer, sauvegarder, exécuter un script (.py)
    • Commentaires avec #
    • Exécuter depuis le terminal : python script.py

Module 2 : Types, texte et structures (10h45 – 12h15)

Objectifs : Identifier les types fondamentaux de Python, maîtriser les structures de données principales.

Contenu détaillé :

  1. Les types numériques (10 min)

    • int (précision arbitraire), float (IEEE 754)
    • Opérateurs : +, -, *, /, //, %, **
    • Conversion : int(), float()

  2. Les chaînes de caractères (15 min)

    • Guillemets simples, doubles, triples
    • f-strings : f"Bonjour {nom}"
    • Méthodes : .upper(), .lower(), .strip(), .split(), .replace(), .join()
    • Indexation et slicing sur les chaînes
    • Immutabilité des chaînes

  3. Booléens et None (10 min)

    • True, False, opérateurs and, or, not
    • Valeurs « falsy » : 0, "", [], {}, None
    • None : absence de valeur
    • Comparaisons : ==, !=, <, >, <=, >=

  4. Les listes (15 min)

    • Création : [], list(), list(range())
    • Mutabilité : modifier, ajouter, supprimer
    • Méthodes : .append(), .extend(), .insert(), .remove(), .pop(), .sort(), .reverse()
    • Opérations : len(), in, concaténation +, répétition *

  5. Indexation et slicing (15 min)

    • Par position : l[0], l[-1]
    • Slicing : l[1:3], l[:3], l[2:], l[::2], l[::-1]
    • Assignation par tranche : l[1:3] = [10, 20]

  6. Les tuples (10 min)

    • Création : (), tuple()
    • Immutabilité : pas de modification après création
    • Unpacking : a, b, c = (1, 2, 3)
    • Quand utiliser un tuple vs une liste

  7. Les dictionnaires (10 min)

    • Création : {}, dict()
    • Accès : d["clé"], d.get("clé", défaut)
    • Méthodes : .keys(), .values(), .items()
    • Itérer sur un dictionnaire

  8. Les ensembles (sets) (5 min)

    • Création : set(), {1, 2, 3}
    • Unicité des éléments
    • Opérations : union |, intersection &, différence -

    • Syntaxe : def nom(arg1, arg2):

def add_item(items: list[str], value: str) -> list[str]:
    # évite de modifier la liste d'origine
    new_items = items.copy()
    new_items.append(value)
    return new_items

Définitions techniques complémentaires (niveau I)

  • Runtime Python : environnement d'exécution qui gère objets, exceptions et appels de fonctions.
  • Binding : association d'un nom Python à une référence d'objet en mémoire.
  • Objet mutable : objet modifiable en place (ex: liste, dictionnaire).
  • Objet immuable : objet non modifiable après création (ex: tuple, chaîne, entier).
  • Slicing : extraction d'une sous-séquence par indices [start:stop:step].
  • Idempotence : propriété d'une opération qui peut être rejouée sans dérive de résultat.
  • Exception : signal d'erreur interrompant le flux normal d'exécution.
  • Traceback : pile d'appels détaillant où et pourquoi une erreur s'est produite.
  • Contrat de fonction : attentes sur les entrées, sorties et erreurs possibles.
  • Encodage : règle de transformation texte↔octets (UTF-8 recommandé).

Détail technique par domaine (niveau I)

Mémoire et références

  • Expliquer la différence entre égalité de valeur (==) et identité d'objet (is).
  • Montrer les effets de bord d'un passage d'objet mutable à une fonction.
  • Utiliser une copie défensive (copy()) quand l'appelant doit conserver son état.

Complexité des opérations courantes

  • Rappeler que l'accès indexé d'une liste est en moyenne O(1), alors que la recherche est O(n).
  • Insister sur l'intérêt des dictionnaires pour les accès par clé en moyenne O(1).
  • Préciser que le tri (sorted) est en O(n log n) et doit être évité dans des boucles inutiles.

Validation et gestion d'erreurs

  • Valider les entrées en début de fonction pour échouer tôt avec un message explicite.
  • Capturer uniquement les exceptions attendues, éviter les except Exception génériques.
  • Chaîner les exceptions avec raise ... from err pour préserver le diagnostic.

Fichiers, formats et encodage

  • Ouvrir les fichiers texte avec encoding="utf-8" pour garantir la portabilité.
  • Séparer la lecture brute, la validation du schéma et la transformation métier.
  • Tracer les lignes rejetées (motif, valeur) pour faciliter la correction des sources.
def parse_amount(raw: str) -> float:
    if raw is None or raw.strip() == "":
        raise ValueError("montant manquant")
    try:
        value = float(raw)
    except ValueError as err:
        raise ValueError(f"montant invalide: {raw!r}") from err
    if value < 0:
        raise ValueError("montant négatif interdit")
    return value
