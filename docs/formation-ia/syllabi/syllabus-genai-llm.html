Formation IA — Syllabus bloc GenAI & LLM

Périmètre : J1 à J5 Durée : 5 jours Focus : LLM, prompts, API, fine-tuning, RAG

Objectif général

Acquérir les bases solides pour concevoir des solutions GenAI utiles en entreprise : choisir le bon modèle, structurer les prompts, intégrer via API et exploiter un corpus documentaire avec RAG.

Programme détaillé

Module 1 — Fondamentaux LLM

Objectif : Comprendre les principes de fonctionnement des LLM et leurs limites opérationnelles.

  • Contenu : tokenisation, contexte, paramètres de génération, limites (hallucinations, biais, coûts).
  • Atelier : analyse de sorties de modèles sur plusieurs cas métier.
  • Résultat attendu : savoir cadrer un usage LLM réaliste et mesurable.

Module 2 — Prompt engineering

Objectif : Concevoir des prompts robustes, testables et réutilisables.

  • Contenu : structure d'instructions, rôle système, zero-shot/few-shot, templates de prompts.
  • Atelier : refonte de prompts instables en prompts industrialisables.
  • Résultat attendu : bibliothèque de prompts versionnés par cas d'usage.

Module 3 — Intégration API LLM

Objectif : Développer une intégration applicative fiable avec gestion de conversation.

  • Contenu : appels API, gestion d'erreurs, retries, streaming, suivi de coût/token.
  • Atelier : implémentation d'un chatbot avec historique de messages.
  • Résultat attendu : prototype fonctionnel exploitable en environnement de test.

Module 4 — Personnalisation des modèles

Objectif : Choisir entre prompting avancé, adaptation de contexte et fine-tuning selon le besoin.

  • Contenu : préparation des données, qualité des exemples, critères de comparaison des approches.
  • Atelier : définition d'un protocole d'évaluation (qualité, coût, latence).
  • Résultat attendu : matrice de décision pour la personnalisation d'un cas réel.

Module 5 — RAG (Retrieval-Augmented Generation)

Objectif : Mettre en place une chaîne RAG complète sur une base documentaire métier.

  • Contenu : chunking, embeddings, indexation vectorielle, retrieval, génération avec citations.
  • Atelier : construction d'un pipeline RAG de bout en bout.
  • Résultat attendu : assistant QA sur documents internes avec réponses traçables.

Compétences visées

  • Analyser un cas métier et choisir une stratégie LLM pertinente.
  • Produire des prompts testables, versionnables et réutilisables.
  • Développer une intégration API robuste avec gestion d'erreurs.
  • Mettre en place un pipeline RAG de production initiale.

Exemples de livrables

  • Guide de prompting interne.
  • Mini chatbot API avec historique et logs.
  • Pipeline RAG complet sur documentation d'entreprise.

Prérequis

  • Python intermédiaire (fonctions, modules, HTTP, JSON).
  • Notions SQL ou manipulation de données textuelles recommandées.
  • Aisance terminal/Git et debug applicatif.
