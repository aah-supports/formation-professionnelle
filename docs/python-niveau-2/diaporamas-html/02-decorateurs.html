Chapitre 2 — Décorateurs

Factoriser les comportements transverses sans alourdir la logique métier.

Objectifs du chapitre

  • Comprendre closures, wrappers et métadonnées de fonction.
  • Créer des décorateurs paramétrables et testables.
  • Décorer correctement des fonctions sync et async.
  • Éviter les pièges de performance et de lisibilité.

Pré-requis conceptuel : fonction objet

def greet(name: str) -> str:
    return f"Bonjour {name}"

fn = greet
print(fn("Ana"))

# Une fonction peut être passée à une autre

def call_twice(f, value):
    return f(value), f(value)

Closure minimale

def make_multiplier(factor: int):
    def inner(x: int) -> int:
        return x * factor
    return inner

x3 = make_multiplier(3)
print(x3(7))  # 21

Structure canonique d'un décorateur

import functools

def log_call(fn):
    @functools.wraps(fn)
    def wrapper(*args, **kwargs):
        print(f"CALL {fn.__name__}")
        result = fn(*args, **kwargs)
        print(f"DONE {fn.__name__}")
        return result
    return wrapper

Pourquoi functools.wraps est obligatoire

  • Conserve __name__, __doc__, annotations et signature.
  • Évite de casser introspection, debug et documentation auto.
  • Facilite les tests et la traçabilité des appels.

Décorateur paramétrable

import functools
import time

def warn_if_slow(threshold_ms: float):
    def deco(fn):
        @functools.wraps(fn)
        def wrapper(*args, **kwargs):
            t0 = time.perf_counter()
            out = fn(*args, **kwargs)
            dt = (time.perf_counter() - t0) * 1000
            if dt > threshold_ms:
                print(f"SLOW {fn.__name__}: {dt:.1f}ms")
            return out
        return wrapper
    return deco

Décorateur d'autorisation métier

def require_role(expected: str):
    def deco(fn):
        def wrapper(user, *args, **kwargs):
            if expected not in user.roles:
                raise PermissionError("accès refusé")
            return fn(user, *args, **kwargs)
        return wrapper
    return deco

Ordre de chaînage des décorateurs

@a
@b
def handler():
    ...

# équivalent à:
handler = a(b(handler))
  • Le décorateur le plus proche de la fonction s'exécute en premier à l'appel.

Décorateur avec état (classe)

class count_calls:
    def __init__(self, fn):
        self.fn = fn
        self.calls = 0

    def __call__(self, *args, **kwargs):
        self.calls += 1
        return self.fn(*args, **kwargs)

Décorer une coroutine async

import functools

def log_async(fn):
    @functools.wraps(fn)
    async def wrapper(*args, **kwargs):
        print(f"ASYNC CALL {fn.__name__}")
        return await fn(*args, **kwargs)
    return wrapper

Retry simple pour I/O instable

import time

def retry(max_attempts: int = 3, delay: float = 0.1):
    def deco(fn):
        def wrapper(*args, **kwargs):
            for attempt in range(1, max_attempts + 1):
                try:
                    return fn(*args, **kwargs)
                except Exception:
                    if attempt == max_attempts:
                        raise
                    time.sleep(delay)
        return wrapper
    return deco

Pièges fréquents

  • Dégrader la lisibilité avec trop de couches décoratives.
  • Masquer des exceptions utiles au diagnostic.
  • Introduire un coût non mesuré dans des chemins chauds.
  • Mélanger validation métier et logique d'infrastructure.

Tester un décorateur

  • Test unitaire de la fonction décorée avec cas nominal + erreur.
  • Vérifier la conservation du nom/annotations/docstring.
  • Tester l'ordre d'exécution si plusieurs décorateurs.
  • Mesurer l'impact perf si utilisé à grande échelle.

Récapitulatif

  • Un décorateur doit rester lisible, ciblé et réutilisable.
  • wraps, tests et contraintes de perf sont indispensables.
  • Les décorateurs structurent bien les préoccupations transverses.
