Chapitre 1 — POO avancée

Structurer une application Python orientée domaine, testable et évolutive.

Compétences visées

  • Modéliser le métier avec des classes cohérentes.
  • Utiliser les méthodes magiques pertinentes.
  • Appliquer composition, Protocol et encapsulation.
  • Préparer le code aux tests et à l'industrialisation.

Objet métier et invariant

from dataclasses import dataclass

@dataclass(slots=True)
class LigneCommande:
    sku: str
    quantite: int
    prix_unitaire: float

    def __post_init__(self) -> None:
        if self.quantite <= 0:
            raise ValueError("quantite doit être > 0")
        if self.prix_unitaire < 0:
            raise ValueError("prix invalide")

Encapsulation avec propriété

class Compte:
    def __init__(self, solde: float = 0.0) -> None:
        self._solde = solde

    @property
    def solde(self) -> float:
        return self._solde

    def debiter(self, montant: float) -> None:
        if montant <= 0 or montant > self._solde:
            raise ValueError("opération refusée")
        self._solde -= montant

Factory avec classmethod

from datetime import datetime

class AuditEvent:
    def __init__(self, actor: str, action: str, at: datetime) -> None:
        self.actor = actor
        self.action = action
        self.at = at

    @classmethod
    def now(cls, actor: str, action: str) -> "AuditEvent":
        return cls(actor=actor, action=action, at=datetime.utcnow())

Méthodes magiques essentielles

  • __repr__ : diagnostic lisible en logs/debug.
  • __eq__ et __hash__ : identité métier explicite.
  • __len__, __iter__, __contains__ : comportement collection.
  • __enter__/__exit__ : gestion de ressources contrôlée.

Objet itérable métier

class Panier:
    def __init__(self, lignes: list[LigneCommande]) -> None:
        self._lignes = lignes

    def __iter__(self):
        return iter(self._lignes)

    def __len__(self) -> int:
        return len(self._lignes)

    def total_ht(self) -> float:
        return sum(l.quantite * l.prix_unitaire for l in self)

Protocol pour réduire le couplage

from typing import Protocol

class PaymentGateway(Protocol):
    def charge(self, amount: float, token: str) -> str: ...

class CheckoutService:
    def __init__(self, gateway: PaymentGateway) -> None:
        self.gateway = gateway

    def pay(self, amount: float, token: str) -> str:
        return self.gateway.charge(amount, token)

Composition plutôt qu'héritage profond

  • La composition isole les responsabilités.
  • L'héritage profond augmente la surface de régression.
  • Préférer des objets collaboratifs courts et spécialisés.
  • Utiliser l'héritage pour un vrai contrat commun stable.

MRO et super()

class Base:
    def log(self):
        print("Base")

class AuditMixin(Base):
    def log(self):
        print("Audit")
        super().log()

class Service(AuditMixin, Base):
    pass

# Service.mro() => Service, AuditMixin, Base, object

Descriptor minimal

class PositiveFloat:
    def __set_name__(self, owner, name):
        self.private = f"_{name}"

    def __set__(self, instance, value):
        if value < 0:
            raise ValueError("valeur négative")
        setattr(instance, self.private, float(value))

Les descriptors permettent de factoriser des règles d'attributs réutilisables.

Exceptions métier explicites

class StockInsuffisantError(Exception):
    pass

class StockService:
    def reserver(self, dispo: int, qte: int) -> None:
        if qte > dispo:
            raise StockInsuffisantError(
                f"demandé={qte} disponible={dispo}"
            )

Checklist de conception POO

  • Une classe = une responsabilité métier principale.
  • Les invariants sont-ils protégés dès la construction ?
  • Les dépendances externes passent-elles par abstractions ?
  • Les comportements critiques sont-ils testables sans I/O ?

Récapitulatif

  • La POO Python efficace repose sur des objets simples et explicites.
  • Méthodes magiques, Protocol et composition améliorent la robustesse.
  • Le design doit d'abord servir la lisibilité et les tests.
