def quality_report(df):
    return {
        "rows": len(df),
        "na_rate": (df.isna().sum() / len(df)).to_dict(),
        "duplicates": int(df.duplicated().sum())
    }

Détails techniques approfondis (niveau III)

Qualité de données

  • Définir des seuils d'acceptation: taux de NA max, taux de doublons max, bornes métier.
  • Standardiser les conventions de nommage de colonnes avant tout enrichissement.
  • Contrôler la stabilité des agrégats d'un run à l'autre (variance anormale).
  • Séparer anomalies bloquantes et anomalies non bloquantes avec journal de décision.

Performance des pipelines

  • Mesurer le coût des étapes critiques (read, merge, groupby, export) sur volume réaliste.
  • Réduire les copies mémoire: éviter chaînes de transformations inutiles et colonnes temporaires superflues.
  • Adapter les dtype pour limiter l'empreinte sans dégrader la précision métier.
  • Découper les traitements volumineux en lots idempotents si nécessaire.

Gouvernance des KPI

  • Documenter source primaire, règles de calcul, exclusions et responsabilité métier.
  • Versionner le calcul d'indicateur pour expliciter les ruptures de série historique.
  • Mettre en place une validation croisée (comptabilité, finance, opérations).
  • Publier avec une fenêtre de recalcul contrôlée pour les données tardives.

Glossaire opérationnel

  • Schema-on-read : interprétation du schéma à la lecture des données.
  • Cardinalité contrôlée : relation de jointure explicitement validée.
  • Rolling window : calcul glissant sur une fenêtre temporelle mobile.
  • Baseline KPI : valeur de référence utilisée pour détecter les écarts.
  • Audit trail : historique horodaté des traitements et décisions.

Définitions techniques complémentaires (niveau III)

  • Data profiling : analyse initiale des distributions, manquants, cardinalités et anomalies.
  • Schema drift : évolution inattendue du schéma source (colonnes, types, formats).
  • Skew : déséquilibre de distribution pouvant dégrader les agrégations ou performances.
  • Partitionnement logique : découpage des traitements par période, région ou domaine métier.
  • Window function : calcul contextuel sur fenêtre ordonnée sans perdre le niveau ligne.
  • Reconciliation : rapprochement entre résultat calculé et source de référence métier.
  • Data quality gate : seuil bloquant empêchant la publication d'un livrable non conforme.

Approfondissement technique (niveau III)

Modélisation et fiabilité des transformations

  • Stabiliser les clés métier en amont pour sécuriser jointures et agrégations.
  • Versionner les règles de transformation pour expliquer chaque rupture de série.
  • Séparer clairement logique de calcul, contrôles qualité et publication des outputs.

Performance et passage à l'échelle

  • Mesurer CPU, mémoire et temps sur des échantillons représentatifs avant optimisation.
  • Réduire les conversions de type répétées en consolidant le typage dès l'ingestion.
  • Découper les pipelines en étapes idempotentes pour reprise partielle après incident.

Qualité de restitution métier

  • Associer chaque KPI à une fiche de définition, propriétaire et règle de recalcul.
  • Afficher systématiquement période couverte, date de calcul et taux de données rejetées.
  • Réaliser une validation croisée avec au moins une source externe de contrôle.
