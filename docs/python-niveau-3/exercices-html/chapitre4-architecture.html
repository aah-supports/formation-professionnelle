Exercice 4.2 — Port / adapter pour dépôt Avancé

  1. Définir un protocole KPIRepository.
  2. Coder un use case ComputeRegionKPI dépendant du protocole.
  3. Créer un adapter concret PandasKPIRepository.
Corrigé 
from typing import Protocol

class KPIRepository(Protocol):
    def sales_by_region(self, month: str) -> list[dict]: ...

class ComputeRegionKPI:
    def __init__(self, repo: KPIRepository) -> None:
        self.repo = repo

    def execute(self, month: str) -> list[dict]:
        rows = self.repo.sales_by_region(month)
        return sorted(rows, key=lambda r: r["ca_total"], reverse=True)

Exercice 4.3 — ADR de décision d'architecture Avancé

Rédiger un mini ADR (1 page max) sur le choix: « pandas local vs entrepôt SQL pour les KPI mensuels ».

  • Contexte
  • Décision
  • Alternatives rejetées
  • Conséquences techniques
Corrigé (trame) 
# ADR-004: Stockage KPI mensuels
Date: 2026-04-22

Contexte:
- Volumes en hausse, recalculs fréquents, besoin d'audit.

Décision:
- Calcul en SQL + extraction pandas pour restitution.

Alternatives:
- Tout en pandas local (rejeté: traçabilité limitée).
- Tout en API externe (rejeté: dépendance forte).

Conséquences:
+ Requêtes reproductibles, gouvernance améliorée.
- Besoin de compétences SQL et monitoring BDD.
