Chapitre 5 — Qualité, typing et tests

Objectif: sécuriser les comportements critiques avec contrats de types et tests ciblés.

Exercice 5.1 — Typage strict d'un service KPI Intermédiaire

  1. Ajouter des annotations complètes sur fonctions publiques.
  2. Créer des TypedDict pour les lignes d'entrée.
  3. Passer un contrôle mypy --strict sur le module.
Corrigé 
from typing import TypedDict

class SaleRow(TypedDict):
    region: str
    amount: float

def total_by_region(rows: list[SaleRow]) -> dict[str, float]:
    out: dict[str, float] = {}
    for r in rows:
        out[r["region"]] = out.get(r["region"], 0.0) + r["amount"]
    return out

Exercice 5.2 — Tests unitaires sur cas limites Avancé

  1. Tester des montants négatifs, vides, et extrêmes.
  2. Utiliser pytest.mark.parametrize.
  3. Vérifier les exceptions attendues.
Corrigé 
import pytest

@pytest.mark.parametrize(
    "amounts,expected",
    [([10.0, 20.0], 30.0), ([0.0], 0.0), ([], 0.0)],
)
def test_sum_amounts(amounts, expected):
    assert sum_amounts(amounts) == expected


def test_sum_amounts_rejects_negative():
    with pytest.raises(ValueError):
        sum_amounts([10.0, -2.0])

Exercice 5.3 — Contrôle qualité PR Avancé

Créer une checklist de revue minimum pour les PR du projet:

  • Types vérifiés
  • Tests passants
  • Cas limites couverts
  • Impact perf explicité
  • Changelog ou note de version
Corrigé (template) 
## Quality Gate PR
- [ ] mypy --strict OK
- [ ] pytest -q OK
- [ ] Cas limites couverts (vide, null, extrêmes)

- [ ] Documentation / changelog mis à jour
